🎒 What to Bring

💻 Software to Install NOW

⚙️ Project Setup (First 30 Minutes)

Step 1: Create Project Folder

# Open Terminal/Command Prompt
cd Desktop
mkdir devclash-project
cd devclash-project

💡 Or just create a folder "devclash-project" on Desktop using mouse!

Step 2: Initialize Git

git init
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

💡 Replace "Your Name" and email with your actual details

Step 3: Create Files

# Windows:
type nul > index.html
type nul > style.css
type nul > script.js
type nul > README.md

# Mac/Linux:
touch index.html style.css script.js README.md

💡 Or create files in VS Code: File → New File

Step 4: First Commit

git add .
git commit -m "Initial commit - created project structure"
✅ Setup Complete! You're ready to start coding.

💻 Hourly Workflow

Hour 0

🏁 Arrival & Setup

  • Plug in laptop, charger, LAN converter
  • Test internet connection
  • Open VS Code, browser, group chat
Hour 1

🧠 Brainstorm

  • What problem will you solve?
  • What tech do you know?
  • Pick ONE simple idea
  • Divide tasks among team
Golden Rule: Simple working project > Complex broken project
Hour 2-6

⚡ Build Core Features

  • Build MAIN feature first (even if ugly)
  • Test every hour
  • Commit to Git every hour
  • Don't add extra features yet!
# Every hour do this:
git add .
git commit -m "Added [feature name]"
git push
Hour 7-8

🎨 Make It Look Good

  • Add CSS styling
  • Use colors, fonts, spacing
  • Make it mobile-responsive
Hour 9-10

🧪 Test & Debug

  • Click EVERY button
  • Test on different browsers
  • Try to break it (enter weird data)
  • Fix critical bugs only
Hour 11

🎤 Prepare Presentation

  • Create slides (Google Slides/PPT)
  • Practice demo 3 times
  • Prepare for Q&A
Hour 12

🚀 Submit!

  • Final Git commit & push
  • Take screenshots
  • Update README.md
  • Submit before deadline!

🌿 Git Commands Cheat Sheet

Basic Commands (Use These!)

git status

Check what changed

git add .

Stage all changes

git commit -m "message"

Save changes

git push

Upload to GitHub

git log

View history

git diff

See what changed

The Git Workflow (Repeat Every Hour)

1
Code for 45 minutes

Write your features in VS Code

2
Save & Test (10 minutes)

Test if everything works

3
Commit (5 minutes) git add . && git commit -m "description" && git push

🆘 Emergency Commands

# Undo changes to a file
git restore filename.html

# Undo last commit (keep changes)
git reset HEAD~1

# See what changed in a file
git diff filename.html

# Go back to previous version
git log # Find commit ID
git checkout [commit-id]
⚠️ Common Mistake: Forgetting to commit regularly. Commit EVERY HOUR minimum!

🌐 How to Run Your Project Locally

🏆 Method 1: VS Code Live Server (RECOMMENDED)

1

Install "Live Server" extension in VS Code

2

Open your project folder in VS Code

3

Right-click index.html → "Open with Live Server"

4

Browser opens at http://localhost:5500 ✅

Magic: Any change you make auto-refreshes the browser!

🐍 Method 2: Python Server

# Navigate to project folder
cd Desktop/devclash-project

# Start server
python -m http.server 8000
# OR
python3 -m http.server 8000

# Open browser at:
# http://localhost:8000

💡 Stop server: Press Ctrl + C

📱 Let Teammates Access Your Site

1

Find your IP address

ipconfig (Windows) | ifconfig (Mac/Linux)
2

Start server (using any method above)

3

Share this URL: http://YOUR-IP:8000

Example: http://192.168.1.45:8000

Note: Both devices must be on SAME WiFi network

💡 Pro Tips from Hackathon Winners

🎯

Start Simple

Build 1 working feature > 10 broken features. Complexity kills projects.

💾

Commit Often

Save to Git every hour. When laptop crashes at 3 AM, you'll thank yourself.

😴

Sleep 4+ Hours

Tired brain codes badly. Quick nap > energy drinks.

🤝

Help Others

Talk to other teams. Steal ideas. Share solutions. Make friends!

⏰

Time Management

Leave 2 hours buffer for bugs and presentation. Demos always break.

😊

Smile During Demo

Even if everything breaks, present confidently. Judges love effort.

📸

Take Screenshots

If demo fails, you have backup screenshots to show.

🔍

Google is Your Friend

90% of problems already solved on Stack Overflow.

🚨 When Everything Breaks (2 Hours Before Deadline)

Step 1: STAY CALM

Panicking wastes time. Breathe.

Step 2: Restore Last Working Version git log → git checkout [old-commit-id]
Step 3: Cut Features

Show what DOES work. 1 working feature > 5 broken.

Step 4: Be Honest

"We faced [challenge], here's what we learned" - Judges respect honesty.
