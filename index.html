→
With Git

✅ One folder
✅ All versions saved

🔁 Core Workflow (The Loop You'll Repeat 100 Times)

STEP 1
git init

Start tracking this project

⚡ Do once per project
STEP 2
git status

What files changed?

🔄 Do often to check
STEP 3
git add .

Prepare ALL changes to save

🔄 Before every commit
STEP 4
git commit -m "message"

Save a snapshot with description

🔄 Every hour minimum!
STEP 5
git push

Upload to GitHub (backup)

🔄 After every commit

🧠 Real Example - You Add a Button

# You just added a login button to index.html # Now save this change: git add . git commit -m "Added login button" git push # Done! This version is saved forever.

🚨 Emergency Commands (When You Mess Up)

😱 Accidentally changed file

git restore file.js

Undo ALL changes to that file

😱 Last commit was wrong

git reset HEAD~1

Undo last commit (keeps changes)

😱 Want to see what changed

git diff

Shows exact changes you made

😱 Everything is broken!

git log
git checkout [old-commit-id]

Go back to when it worked

🔐 Security Basics (.env & .gitignore)

🚨 CRITICAL: Leak your API keys = Hackers can use your account = You get charged!

🔐 What is .env File?

Purpose: Store secret data that should NEVER be shared

# Example .env file API_KEY=abcd1234xyz DB_PASSWORD=super_secret_password JWT_SECRET=my_secret_token STRIPE_KEY=sk_test_123456

❌ WRONG vs ✅ RIGHT

❌ NEVER DO THIS

// script.js const API_KEY = "abcd1234xyz"; fetch(url, { key: API_KEY });

👆 Anyone can see this in your code!

✅ DO THIS INSTEAD

// .env file API_KEY=abcd1234xyz // script.js const API_KEY = process.env.API_KEY; fetch(url, { key: API_KEY });

👆 Secret stays hidden!

🚫 What is .gitignore?

Purpose: Tell Git which files to NEVER upload to GitHub

# Example .gitignore file # Never upload these: .env node_modules/ dist/ *.log .DS_Store # Why? # .env → has secrets # node_modules → too large (1000s of files) # dist → auto-generated # *.log → debug files

📋 What to Ignore (Common List)

.env Secret keys & passwords
node_modules/ Downloaded libraries (too large)
dist/ Auto-generated build files
*.log Debug logs
.DS_Store Mac system file (useless)
⚡ PRO TIP: Create .gitignore BEFORE first commit! Add .env to it immediately!

🧩 How to Read Huge AI Code (Game Changer)

😱 The Problem

You ask ChatGPT for code.

It gives you 1000+ lines across 10 files.

You stare at it. Your brain freezes. 🧊

👉 This section will FIX that!

❌ DON'T Do This

❌ Try reading every line
❌ Start from top
❌ Understand everything first
❌ Give up and refresh ChatGPT

✅ DO This Instead (4-Step System)

STEP 1

Find the Entry Point

Look for these files first:

  • index.html - For websites
  • index.js or app.js - For Node.js
  • main.py - For Python
  • App.jsx - For React

👉 This is where the program STARTS

STEP 2

Find the Flow

Look for these keywords:

routes function fetch onClick addEventListener API

👉 These show HOW things connect

STEP 3

Use Search (Ctrl + F)

Search for what YOU care about:

  • Building login? → Search "login"
  • Want to see API calls? → Search "fetch"
  • Looking for database? → Search "db" or "database"

👉 Jump straight to relevant code!

STEP 4

Break Into Chunks

Treat code like sections:

UI

How it looks

Logic

How it works

API

How it talks to backend

👉 Understand ONE chunk at a time

🧪 Real Example

Scenario: ChatGPT gave you a todo app with 800 lines

Step 1: Open index.html
Step 2: Press Ctrl + F → search "todo"
Step 3: Find addTodo() function
Step 4: Read just that function (ignore rest)
Step 5: Run the code, see what addTodo() does
Result: You understand ONE feature! Repeat for others.
💡 PRO TIP: Add console.log() everywhere to see what's happening!

🌐 How to Run Your Project Locally

🏆 Method 1: VS Code Live Server (BEST!)

1

Install "Live Server" extension in VS Code

2

Open your project folder

3

Right-click index.html → "Open with Live Server"

4

Browser opens at http://localhost:5500 ✅

Magic: Auto-refreshes on every save!

🐍 Method 2: Python Server

# Navigate to project cd Desktop/hackathon-project # Start server python -m http.server 8000 # Open browser: # http://localhost:8000 # Stop: Ctrl + C

📱 Share with Teammates (Same WiFi)

  1. Find your IP: ipconfig (Windows) or ifconfig (Mac)
  2. Start server (any method)
  3. Share: http://YOUR-IP:8000
  4. Example: http://192.168.1.45:8000
Note: Both must be on SAME WiFi!
