🔁 The Core Workflow (Repeat 100x)

This is the loop you'll do EVERY HOUR during hackathon:

STEP 1 - Do Once
git init

Start tracking this project

STEP 2 - Check Often
git status

What files changed?

STEP 3 - Before Commit
git add .

Prepare ALL changes (the dot means "everything")

STEP 4 - Every Hour!
git commit -m "Added login button"

Save snapshot with description of what you did

STEP 5 - After Commit
git push

Upload to GitHub (cloud backup!)

🧠 Real Example - You Add a Button

# You just added a login button to index.html # Now save this change: git add . git commit -m "Added login button" git push # Done! This version is saved forever. # Even if laptop explodes, it's on GitHub!

🚨 Emergency Commands (When You Mess Up)

😱 Accidentally changed file

git restore file.js

Undo ALL changes to that file

😱 Last commit was wrong

git reset HEAD~1

Undo last commit (keeps changes)

😱 Want to see changes

git diff

Shows exact changes you made

😱 Everything is broken!

git log git checkout [commit-id]

Go back to when it worked

⚡ GOLDEN RULE: Commit EVERY HOUR during hackathon! Set a timer if you have to. Git = Your time machine!
