📂 Level 5: File Structure

Organize your code like a pro!

🔥 TRUTH: Bad file structure = You'll get lost in your own code after 2 hours!

🏗️ Basic Structure (For Simple Projects)

Perfect for hackathons and small projects:

📁 project/
📄 index.html ← Main page
🎨 style.css ← Design
⚡ script.js ← Functionality
📝 README.md ← Project info
🚫 .gitignore ← Files to ignore

🏗️ Advanced Structure (With Frontend/Backend)

When you have separate frontend and backend:

📁 project/
📁 frontend/
📄 index.html
🎨 style.css
⚡ script.js
📁 backend/
🐍 server.py ← Python server
🟢 app.js ← Node.js server
🔐 .env ← Secret keys
🚫 .gitignore
📝 README.md

💡 When You See Confusing Folders in AI Code

Don't panic! Here's what they mean:

src/ Source

Main code lives here

components/ Reusable Parts

Buttons, cards, etc.

utils/ Utilities

Helper functions

services/ API Calls

Talks to backend

node_modules/ Libraries

Downloaded packages (DON'T TOUCH!)

dist/ Distribution

Final built version

👉 Pro Tip: When confused, look for index.html, app.js, or main.py - that's where the program starts!
