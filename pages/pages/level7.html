🔐 Level 7: Security Secrets

Protect your API keys and passwords!

🚨 CRITICAL WARNING: Leak your API keys = Hackers can use your account = You get CHARGED thousands of dollars! This is NOT a joke!

🔐 What is .env File?

Purpose: Store secret data that should NEVER be shared publicly

# Example .env file # This file should NEVER go to GitHub! API_KEY=abcd1234xyz_super_secret DB_PASSWORD=my_database_password_123 JWT_SECRET=token_secret_key STRIPE_KEY=sk_test_123456789 OPENAI_KEY=sk-proj-abcdefgh

❌ WRONG vs ✅ RIGHT

❌ NEVER DO THIS

// script.js const API_KEY = "abcd1234xyz"; fetch(url, { headers: { 'Authorization': API_KEY } });

⚠️ Anyone can see this in your code!
⚠️ If you push to GitHub = EXPOSED!
⚠️ Hackers WILL find and use it!

✅ DO THIS INSTEAD

// .env file (NEVER commit this!) API_KEY=abcd1234xyz // script.js const API_KEY = process.env.API_KEY; fetch(url, { headers: { 'Authorization': API_KEY } });

✅ Secret stays in .env file
✅ .env is ignored by Git
✅ Safe and secure!

🚫 What is .gitignore?

Purpose: Tell Git which files to NEVER upload to GitHub

# .gitignore file # Create this in your project root # Secret files .env .env.local config.json # Dependencies (too large!) node_modules/ venv/ __pycache__/ # Build files (auto-generated) dist/ build/ *.log # OS files (useless) .DS_Store Thumbs.db # IDE files .vscode/ .idea/

📋 What to Ignore (Common Files)

.env Secret keys & passwords
node_modules/ Downloaded libraries (HUGE!)
dist/ Auto-generated build files
*.log Debug logs (useless)
.DS_Store Mac system file (junk)

🛡️ Security Checklist for Hackathon

Create .env file for ALL secrets
Create .gitignore BEFORE first commit
Add .env to .gitignore
NEVER commit API keys in code
Check GitHub after pushing (make sure .env not there!)
⚡ GOLDEN RULE: If it's SECRET, it goes in .env. If it's in .env, it goes in .gitignore. NO EXCEPTIONS!

🚨 What If You Already Leaked a Key?

Emergency Steps:

  1. IMMEDIATELY regenerate the key (go to API provider dashboard)
  2. Delete the key from your code
  3. Move it to .env file
  4. Add .env to .gitignore
  5. Git commit with new setup

Note: Simply deleting from GitHub is NOT enough! The key is in commit history. You MUST regenerate it!
